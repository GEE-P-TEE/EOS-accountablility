# Accountability Chart Maker - Master Plan

## App Overview and Objectives

The Accountability Chart Maker is a comprehensive web-based platform designed specifically for business owners implementing the Entrepreneurial Operating System (EOS) methodology. This application streamlines the creation, management, and sharing of visual organizational charts that clearly define roles, responsibilities, and accountability structures within organizations.

### Primary Objectives
- Simplify EOS accountability chart creation through an intuitive drag-and-drop interface optimized for web browsers
- Enable seamless real-time collaboration between team members and stakeholders
- Provide professional-grade export capabilities for presentations, documentation, and sharing
- Offer comprehensive pre-built templates tailored to ecommerce, Amazon sellers, and various business types
- Maintain enterprise-level data integrity and security for sensitive organizational information
- Create an intuitive, collaborative platform that feels professional while remaining accessible

## Target Audience

### Primary Users
- **Business Owners Implementing EOS**: Entrepreneurs and executives adopting EOS methodology who need to visualize their organizational structure
- **EOS Implementers**: Consultants and coaches helping businesses implement EOS frameworks
- **Team Leaders**: Department heads and managers responsible for defining team structures

### Secondary Users
- **Ecommerce Business Owners**: Online retailers organizing marketing, operations, and fulfillment teams
- **Amazon Sellers**: Marketplace sellers structuring PPC, inventory, and customer service roles
- **Small to Medium Business Leaders**: Companies with 10-250 employees seeking organizational clarity

## Core Features and Functionality

### Chart Creation and Management
- Interactive drag-and-drop chart builder with visual hierarchy
- Seat/role cards containing role names, person assignments, responsibilities, and measurables
- Dynamic add/remove functionality for organizational positions
- Auto-save capabilities with cloud-based storage
- Customizable styling options including colors and layout preferences

### Collaboration Features
- Real-time multi-user editing capabilities
- Team member invitation and permission management
- Comment system for role-specific feedback and discussions
- Activity logging for change tracking and accountability
- Role-based access controls (view-only, editor, admin)

### Template System
- Pre-built organizational structures for various business types
- Industry-specific templates (ecommerce, Amazon sellers, service businesses)
- Custom template creation and sharing capabilities
- Template categorization and search functionality

### Export and Sharing
- Professional PDF generation with EOS-style formatting
- Shareable read-only links with customizable access permissions
- Print-optimized layouts for physical documentation
- Multiple image export formats (PNG, SVG)

## High-Level Technical Stack Recommendations

### Frontend Architecture
- **Framework**: Next.js 14 with App Router for optimal performance and SEO
- **Styling**: Tailwind CSS for utility-first responsive design
- **UI Components**: shadcn/ui for consistent, accessible component library
- **Drag-and-Drop**: React DnD for intuitive chart manipulation
- **State Management**: Zustand or React Context for application state

### Backend and Database
- **Backend-as-a-Service**: Supabase for authentication, database, and real-time features
- **Database**: PostgreSQL (via Supabase) for relational data management
- **Real-time**: Supabase real-time subscriptions for collaborative editing
- **File Storage**: Supabase Storage for chart exports and user assets

### Additional Services
- **Authentication**: Supabase Auth with email/password and Google OAuth options
- **PDF Generation**: jsPDF or Puppeteer for high-quality document exports
- **Content Management**: Notion API for help documentation and template management
- **Hosting**: Vercel for seamless Next.js deployment and global CDN

## Conceptual Data Model

### Core Entities
- **Users**: Authentication, profile information, subscription status
- **Charts**: Chart metadata, ownership, creation/modification timestamps
- **Seats**: Individual positions within charts including role details and positioning
- **Measurables**: Key performance indicators and targets associated with seats
- **Templates**: Pre-built chart structures with categorization
- **Collaborators**: User permissions and access levels for shared charts

### Relationships
- Users can own multiple Charts (one-to-many)
- Charts contain multiple Seats (one-to-many)
- Seats can have multiple Measurables (one-to-many)
- Charts can have multiple Collaborators (many-to-many through junction table)
- Templates can be used to create multiple Charts (one-to-many)

## User Roles and Access Rights

### 1. Chart Owner (Creator)
**Full Administrative Access**:
- Create, edit, and delete charts
- Manage all collaboration settings
- Invite and remove team members
- Set permission levels for collaborators
- Export charts in all available formats
- Access to usage analytics and activity logs
- Ability to transfer ownership

### 2. Chart Admin (Delegated Authority)
**Administrative Access with Limitations**:
- Edit chart content and structure
- Invite new collaborators (with owner approval)
- Manage comments and feedback
- Export charts in standard formats
- View activity logs and collaboration history
- Cannot delete charts or remove owner access

### 3. Chart Editor (Active Collaborator)
**Content Creation and Modification**:
- Edit seat information and organizational structure
- Add and modify measurables and responsibilities
- Participate in comment discussions
- View real-time changes from other collaborators
- Export charts for personal use
- Cannot manage team members or sharing settings

### 4. Chart Viewer (Read-Only Access)
**View-Only Permissions**:
- View chart content and structure
- Access to comments and discussions (read-only)
- Print and basic export functionality
- Cannot make any modifications to chart content
- Cannot invite additional users

### 5. Public Viewer (Shared Link Access)
**Limited Anonymous Access**:
- View shared charts through public links
- No account required for access
- Cannot comment or interact with content
- Limited export options (if enabled by owner)
- Session-based access without persistent permissions

## User Flow Patterns

### New User Onboarding Flow
1. **Landing Page** → Sign up with email or Google OAuth
2. **Email Verification** → Account activation
3. **Welcome Tutorial** → Brief platform introduction
4. **Template Selection** → Choose starting point or blank canvas
5. **Chart Builder** → Guided chart creation with tooltips
6. **First Export/Share** → Complete initial workflow
7. **Dashboard** → Return to main hub for ongoing use

### Returning User Flow
1. **Login** → Authentication
2. **Dashboard** → Chart overview and recent activity
3. **Chart Selection** → Open existing chart or create new
4. **Chart Builder** → Edit and collaborate
5. **Export/Share** → Distribute completed charts
6. **Dashboard** → Return to main hub

### Collaboration Flow
1. **Chart Owner** → Creates chart and invites collaborators
2. **Email Invitation** → Team members receive access invitation
3. **Account Creation** → New users sign up (if needed)
4. **Chart Access** → Collaborators join chart workspace
5. **Real-time Editing** → Multiple users edit simultaneously
6. **Comment and Review** → Feedback and discussion workflow
7. **Final Export** → Owner exports completed collaborative chart

### Template Usage Flow
1. **Templates Library** → Browse available templates
2. **Template Preview** → Examine structure and content
3. **Template Selection** → Choose appropriate template
4. **Chart Customization** → Modify for specific business needs
5. **Team Addition** → Add real people and data
6. **Collaboration** → Share with team for input
7. **Finalization** → Export and distribute final chart
